Update History

  

V2.5.0 / 14-October-2013

 

Main Changes

•  Suppression of STM8L devices with charge-transfer (CT) acquisition.

   Note: These devices and the CT acquisition are now supported in the new STMTouch driver.

•  Only STM8S/A devices with RC acquisition are now supported in this driver.

•  Driver name changed from "STM8 TouchSensing Driver" to "STM8S TouchSensing Driver".

 

Development Toolchains and Compilers

•  ST Visual Develop (STVD) v4.3.4.

•  Cosmic STM8 C Compiler v4.3.9.

•  Raisonance Ride7/RKit-STM8 v2.51.13.

•  IAR Embedded Workbench for STM8 v1.41.

 

Known Limitations

V2.3.0 / 15-September-2010

 

Main Changes

•  To port an application from STM8 Touch Sensing Library V2.2.0 to V2.3.0, the user must create again his tsl_conf.h file from a tsl_conf_RC_TOADAPT.h or tsl_conf_CT_TOADAPT.h V2.3.0 file with its own parameters :

º  PORT MASK definition for multi-channel keys section has been removed (the preprocessor variable definitions are automatically created in stm8_tsl_check_config.h).

º  Preprocessor variables :

–  PROTECT_IO_ACCESS: allows to protect the IO register modifications by interruption subroutines during accesses from Touch Sensing acquisition functions, (to be kept at 1 to keep same behaviour as V2.2.0).

–  SW_SPREAD_SPECTRUM : for STM8L15x only, allows to use the same Spread Spectrum mechanism as implemented for the STM8L10x in order to keep a constant internal osillator frequency for USART communication for example, (0 to disable the feature).

–  SPREAD_COUNTER_MAX is now used for STM8L15x hardware spread spectrum excursion (if SPREAD_SPECTRUM set to 1),

–  ACTIVE_SHIELD_GROUP : defines the switch group dedicated to the active shield feature (0 to disable the feature).

–  RTOS_MANAGEMENT : allows to call the timebase routine from the RTOS instead of using a timer interrupt every 500µs,

–  TIMER_CALLBACK : allows to call a function from the timer interrupt used as 500µs timebase,

–  USE_INLINED_FUNCTIONS : forces inlining of the SW spread spectrum function only (more function may be inlined in the future). No effect with IAR.

º  All the preprocessor variables must be defined but set to 0 to disable the associated feature.

º  Support of IAR C compiler: Tested with v1.10.0.

º  Add active shield feature for charge-transfer acquisition principle.

º  Add callback capability for time-base interrupt sub-routine.

º  Add RTOS management capability of the time-base function.

º  Add forced inlining option for the software spread spectrum function.

º  Add software spread spectrum for STM8L15x.

º  Add amplitude management of the hardware spread spectrum for STM8L15x.

º  Add the ability to define an application with multi-channel keys but without single channel key.

–  Add a new compilation variable automatically generated : NUMBER_OF_ACQUISITION_PORTS.

º  Add the ability to skip the port acquisition when all its channels are disabled.

º  Improve acquisition speed for charge transfer acquisition for both STM8L10x and STM8L15x.

º  Improve the management of the disabled channels during acquisition.

º  Add the management of the interrupt masking defined by a compilation variable.

º  Remove calibration related functions from the API.

º  Remove alignment mechanism for assembly functions of RC acquisition principle file.

º  Cleaning RC acquisition principle related code in charge transfer acquisition principle dedicated files.

º  Improve checking performed in the stm8_tsl_check_config.h file.

º  Simplify the multi-channel key definitions by automatically defining the MKEY_CHi_MASK compilation variable from stm8_tsl_check_config.h file.

º  CLWHTA and CLWLTA are now also used by STM8L10x related functions for charge-transfer acquisition principle.

º  Fix the adjustment of the low state pulse for STM8L10x charge-transfer acquisition.

º  Rename Detection exclusion system acronym from DES to DxS.

  

V2.2.0 / 30-November-2009

 

Main Changes

•  New folders and workspace architecture.

•  Support of Raisonance C compiler:

º  Full compiler version required for inline assembly support.

º  Tested with v2.26.09.0317.

•  ECS limitation corrected:

º  References were frozen for 2 seconds whenever entering in post or pre states.

º  This was problematic in noisy environment.

•  STM8L15x Charge Transfer acquisition optimized (30% faster).

•  New parameters for STM8L15x introduced in stm8_tsl_conf_CT.h file for capacitance charging/discharging tuning:

º  CLWHTA (Capacitance Loading Waiting HIGH Time Adjustment).

º  CLWLTA (Capacitance Loading Waiting HIGH Time Adjustment).

  

V2.1.0 / 30-September-2009

 

Main Changes

•  Source files clean-up, MISRA check.

•  Update to latest STM8L standard peripheral libraries (STM8L10x v1.1.0 and STM8L15x v1.0.0).

V2.0.0 / 31-August-2009

 

Main Changes

•  Update to latest standard peripheral libraries.

V1.3.0 / 29-October-2009

 

Main Changes

•  Update to latest STM8S Standard Library V1.1.1.

•  Update acquisition and timebase timers table of STM8_TSL_RC_Configuration.h file.

•  Change condition to disable ECS when a SCKey or a MCKey is touched.

  

V1.2.0 / 24-March-2009

 

Main Changes

•  Add an option in the configuration file to choose the acquisition timer. This option allows to use the library with any STM8 device.

•  Add a synchronization capability (beta) on an interruption. This option is new and has not been fully validated.

•  Add Spread spectrum capability (beta). This option allows adding a variable delay between acquisitions to reduce noise sensitivity at specific frequencies. This option is new and has not been fully validated.

Main Changes

•  New option (NEGDETECT_AUTOCAL) added in the configuration file to enable or disable the autocalibration.
